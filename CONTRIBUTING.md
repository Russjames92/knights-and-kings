# Contributing

## Branch naming

Use the following prefixes:

- `feature/` for new functionality
- `fix/` for bug fixes
- `chore/` for maintenance or tooling updates
- `docs/` for documentation updates

## Pull request rules

- All changes must go through a pull request.
- Keep PRs focused and scoped to one change set.
- Ensure required status checks pass before requesting review.
- Include a summary and testing notes in the PR description.
